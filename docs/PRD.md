## 사워도우 베이킹 도우미 앱: 제품 요구사항 명세서 (PRD)

### 1. 문서 개요

본 문서는 '사워도우 베이킹 도우미' 애플리케이션 개발을 위한 기능, 사용자 경험(UX), 시스템 요구사항을 정의하는 것을 목적으로 합니다. `docs/sample-web.html` 파일의 내용을 기반으로 핵심 기능을 도출하고, 사용자의 성공적인 베이킹 경험을 돕기 위한 요구사항을 기술합니다.

*   **프로젝트 목표:** 사워도우 베이킹 과정의 복잡한 계산을 자동화하고, 관련 정보를 쉽게 찾아볼 수 있는 통합 환경을 제공하여 모든 수준의 베이커가 더 즐겁고 성공적인 베이킹을 할 수 있도록 돕는다.

### 2. 사용자 분석 (User Personas)

*   **초보 베이커:** 사워도우 베이킹을 이제 막 시작하여 레시피의 '피딩 비율'이나 '베이커스 퍼센트' 같은 개념이 낯선 사용자. 정확한 계산과 상세한 가이드가 필요하다.
*   **중급 베이커:** 기본적인 베이킹은 가능하지만, 다양한 레시피를 시도하며 결과물의 일관성을 높이고자 하는 사용자. 레시피에 따른 정확한 재료 계산 기능이 중요하다.
*   **숙련 베이커:** 자신만의 레시피를 개발하고 베이킹 과정을 최적화하려는 사용자. 다양한 비율을 실험하고 결과를 예측하는 데 계산기를 유용하게 사용한다.

### 3. 핵심 기능 명세

#### 3.1. 사워도우 계산기 (Sourdough Calculator)

*   **설명:** 베이킹에 필요한 스타터, 밀가루, 물의 양을 '비율' 또는 '시간' 기준으로 계산하는 핵심 기능입니다.
*   **요구사항:**
    1.  **계산기 모드 전환:** 사용자는 '비율 기반'과 '시간 기반' 두 가지 모드를 명확하게 선택하고 전환할 수 있어야 합니다.
    2.  **비율 기반 계산:**
        *   **입력:** 총 스타터 양(g), 스타터 비율, 밀가루 비율, 물 비율
        *   **로직:** `(총 스타터 양 * 각 재료 비율) / 전체 비율의 합` 공식을 사용하여 각 재료의 양을 계산합니다.
        *   **출력:** 필요한 스타터, 밀가루, 물의 최종 양(g)을 정수로 반올림하여 표시합니다.
    3.  **시간 기반 계산:**
        *   **입력:** 총 스타터 양(g), 목표 시간 범위 (예: "6-8시간")
        *   **로직:** 선택된 시간 범위에 미리 정의된 피딩 비율(예: "6-8시간" -> 1:2:2)을 자동으로 적용하여 재료의 양을 계산합니다.
        *   **출력:** 계산된 스타터, 밀가루, 물의 최종 양(g)을 표시합니다.
    4.  **결과 표시:** 계산 결과는 사용자가 쉽게 인지할 수 있도록 명확하고 큰 텍스트로 구분되어 표시되어야 합니다.

#### 3.2. 레시피 및 가이드 (Recipe & Guide)

*   **설명:** `sample-web.html`에 포함된 사워도우 관련 교육 콘텐츠를 앱 내에서 쉽게 열람할 수 있는 기능입니다.
*   **요구사항:**
    1.  **콘텐츠 분류:** '피딩 비율', '베이커스 퍼센트', '용어 설명' 등 주제별로 콘텐츠를 분류하여 리스트 형태로 제공합니다.
    2.  **상세 보기:** 리스트에서 특정 항목을 선택하면 해당 주제의 상세 설명 페이지로 이동합니다.
    3.  **검색 기능:** 사용자가 키워드를 입력하여 원하는 정보(예: "후치", "hooch")를 빠르게 찾을 수 있어야 합니다.

### 4. 화면 레이아웃 및 UX (Wireframe/Layout)

*   **전체 구조:** 하단 탭 네비게이션을 사용하여 '계산기'와 '가이드' 두 가지 주요 화면으로 명확히 구분합니다.

*   **화면 1: 계산기 탭**
    *   **상단:** '비율 기반' / '시간 기반' 탭 스위치를 배치하여 두 계산기 간 전환이 용이하도록 합니다.
    *   **중단:** 선택된 모드에 맞는 입력 필드(숫자 입력 칸, 드롭다운 메뉴 등)를 배치합니다.
    *   **하단:** '계산하기' 버튼과, 버튼 아래에 계산 결과가 표시될 결과 영역을 둡니다.

*   **화면 2: 가이드 탭**
    *   **상단:** 콘텐츠 검색을 위한 검색창을 배치합니다.
    *   **중단:** 주제별로 분류된 가이드 목록을 표시합니다. 각 항목은 터치 가능한 버튼 형태로 디자인합니다.

### 5. 비기능적 요구사항

*   **플랫폼:** 크로스플랫폼 모바일 앱 (iOS, Android)
*   **언어:** 한국어
*   **디자인:** 깔끔하고 직관적인 미니멀리즘 디자인을 적용하여 사용성을 높입니다.

### 6. 개발 환경 및 기술 스택

*   **프레임워크:** Flutter - 하나의 코드 베이스로 iOS와 Android 앱을 동시에 개발하기 위해 선택합니다.
*   **디자인 시스템:** Material 3 - 현대적이고 세련된 UI를 위해 Flutter의 최신 표준 디자인 시스템을 사용합니다.
*   **데이터베이스:** SQLite - 사용자의 레시피, 타이머 설정 등 개인화된 데이터를 디바이스에 안전하게 저장하기 위해 로컬 데이터베이스를 사용합니다.
*   **ORM (Object-Relational Mapping):** Drift (구 Moor) - 타입-세이프 코드를 작성하고 SQL 쿼리를 Dart 언어로 쉽게 관리하기 위해 사용합니다.
*   **주요 패키지:**
    *   `sqflite`: Flutter에서 SQLite 데이터베이스를 사용하기 위한 핵심 플러그인.
    *   `drift`: 강력한 반응형 ORM.
    *   `path_provider`: 데이터베이스 파일 저장을 위한 안전한 로컬 경로를 제공.
*   **폴더 구조:**
    *   `lib/database`: 데이터베이스 연결 및 테이블 정의 관련 코드.
    *   `lib/models`: 데이터 모델 클래스.
    *   `lib/screens`: 각 화면을 구성하는 UI 위젯.
    *   `lib/widgets`: 여러 화면에서 공통으로 사용되는 UI 컴포넌트.

### 7. 향후 개선 과제 (Future Roadmap)
